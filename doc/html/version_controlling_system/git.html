    Git是個分散式的版本控制軟體。大部分的版本控制軟體都需要一個伺服端，在commit的同時，提交到伺服器上面。git當然也可以集中管理，但是也多了分散管理的方式。簡單來說，billy開個專案，tom覺得這個專案有意思，他想玩玩看，他可把整個repo(reoisitories) clone，並且在他那邊自己有一個repo。tom可自由的對這個repo做各種提交修改，直到他覺得滿意為止。這時候tom可以告訴billy，他改了一些什麼，若billy覺得不錯，可以從tom那邊fetch回去並merge。
    GitHub是一個大型程式專案平台，奠基於開放原始碼文化，供程式設計師協作與版本控制。由於repo可以綁在寫程式的人身上，不用一定要有中央伺服器的主從架構，因此就出現了類似GitHub的線上版本控制服務。

    Git Server

    若多人共用的話, 要走 SSH 有下述幾種方法:
    使用 git(不使用其他套件)，建立新帳號(git), 由 git 帳號來統一管理(commit / push) 等
    使用 gitosis 來管理(gitosis 可以設定到 "誰" 可以存取此專案).
    使用 gitolite 來管理(gitolite 可以設定 "誰" 可以存取此專案, 而且, 可以設定只能存取哪個 branch 等路徑)

    via SSH(onlt git-core package)

    # 設定帳號, 群組
    Server:~ # groupadd -g 115 git
    Server:~ # useradd -g 115 -u 115 -d /srv/git -s /usr/bin/git-shell -m git
    Server:~ # mkdir ~git/.ssh
    Server:~ # touch ~git/.ssh/authorized_keys # 之後使用者的public key導入此檔案內

    # 建立專案
    Server:~ # mkdir ~git/project.git
    Server:~ # git init --bare --shared project.git/
    Server:~ # chown -R git.git ~git/

    Gitweb

    gitweb是一個git套件(用perl寫的cgi)

    Server:~ # grep projectroot /usr/share/gitweb/gitweb.cgi # 預設讀取git目錄設定
    our $projectroot = "/srv/git";

    Server:~ # cat /etc/apache2/conf.d/gitweb.conf
    Alias /git "/usr/share/gitweb/"
    <Directory "/usr/share/gitweb">
        Options ExecCGI
        AllowOverride None
        AddHandler cgi-script .cgi
        DirectoryIndex gitweb.cgi
        Order allow,deny
        Allow from all

    # 以下設定HTTP Basic Authentication, 若要publish 給任何人可不需要
        AuthType Basic
        AuthName "Git"
        AuthUserFile /srv/git/git.htpasswd
        Require valid-user
    </Directory>

    Server:~ # htpasswd2 -c /srv/git/git.htpasswd user1 # 當檔案不存在時需要加上-c create file
    Server:~ # htpasswd2 /srv/git/git.htpasswd user2

    Git Client

    Basic

    # Configration file
    Client:~ $ git config --system # 使用/etc/gitconfig
    Client:~ $ git config --global # 使用~/.gitconfig
    Client:~ $ git config --file config # 指定git 預設使用.git/config

    # Protocol
    Client:~ $ git clone git@github.com:user/repo.git # SSH protocol, 安全性最佳
    Client:~ $ git clone https://user@github.com/user/repo.git # HTTP/HTTPS protocol, 速度最差, 突破防火牆限制
    Client:~ $ git clone git://github.com/user/repo.git # Git protocol 速度最快, 但缺認證機制(只做唯讀)
    Client:~ $ git clone file://path/repo.git # File 本機目錄 (用 Dropbox 分享 git init –bare –shared 目錄!!)

    # Environment variable
    Client:~ $ git config --global user.name 'owner' # 設定使用者名稱
    Client:~ $ git config --global user.email 'owner@localhost' # 設定email
    Client:~ $ git config --global core.editor vim # 設定文字編輯器
    Client:~ $ git config --global merge.tool vimdiff # 設定比較工具
    Client:~ $ git config --global color.ui true # 設定顯示顏色
    Client:~ $ git config --global apply.whitespace nowarn # 忽略空白的變化, 空白對有些語言是有影響的(像是Ruby)
    Client:~ $ git config --global alias st status # 設定別名, 打 git st 就等同於打 git status
    Client:~ $ git config --list # 顯示當前設定

    Client:~ $ cat ~/.gitconfig
    [user]
        name = owner
        email = owner@localhost
    [core]
        editor = vim
    [merge]
        tool = vimdiff
    [color]
        ui = true
    [alias]
        st = status
        co = checkout
        ci = commit
        br = branch

    Local

    # create new project
    Client:~/project $ git init # 建立git repository, 在project目錄底下產生 .git, 只有第一次建立時需要
    Client:~/project $ ls .git
    HEAD        config      hooks       objects
    branches    description info        refs
    Client:~/project $ cat .git/config # project設定檔
    Client:~/project $ cat .gitignore # project設定不控管檔案格式

    #download project from github/clone repository
    Client:~ $ git clone git@github.com:user/sandbox.git # 下載project
    Client:~ $ git clone git@github.com:user/sandbox.git newproejct # 下載project並改名為newproject

    # commit & branch status:
    Client:~/project $ git add file # 檔案納入控制系統
    Client:~/project $ git rm file # 檔案移出制系統
    Client:~/project $ git commit -m 'initial project version' # 提交到控制系統
    Client:~/project $ git status # version記錄
    Client:~/project $ git log # commit記錄
    Client:~/project $ git log --stat
    Client:~/project $ git log -p
    Client:~/project $ git log --graph --oneline --all # 圖式化顯示

    #first time push
    Client:~/project $ git remote add origin git@github.com:user/repo.git
    Client:~/project $ git push -u origin master # origin是repository, master是branch

    # push & pull
    Client:~/project $ git push # 提交到server
    Client:~/project $ git pull # 從server更新

    #branch
    Client:~/project $ git branch # 顯示分支
    Client:~/project $ git branch b1 # 新增分支
    Client:~/project $ git checkout -b b1 # 新增分支
    Client:~/project $ git branch -d b1 # 刪除分支
    Client:~/project $ git checkout b1 # 切換分支
    Client:~/project $ git fetch
    Client:~/project $ git fetch --all

    Remote

    Client:~/project $ git remote show
    Client:~/project $ git remote set-url origin git@github.com:user/repo.git
    Client:~/project $ git branch -r
    Client:~/project $ git checkout -t
    Client:~/project $ git checkout origin/b1 # 切換到remote bracnh
    Client:~/project $ git push origin b1 # 新增branch到git server
    Client:~/project $ git push origin --delete b1 # 刪除git server上的branch
    Client:~/project $ git push origin :b1 # 刪除git server上的branch
    Client:~/project $ git fetch -p #

    tag

    Client:~/project $ git tag -l [tar_name] # 顯示tag名稱
    Client:~/project $ git tag -a tag_name -m 'comment' [object] # 新增tag
    Client:~/project $ git show tag_name # 顯示tag 內容
    Client:~/project $ git push origin tar_name # 上傳tag
    Client:~/project $ git push origin --tags # 上傳多個tag
    Client:~/project $ git tag -d tag_name # 刪除tag
    Client:~/project $ git push origin :refs/tags/tag_name # 刪除remote tag
